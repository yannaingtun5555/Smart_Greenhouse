Runtime layout:

1. Local Docker uses `server/docker-compose.yml` and the standalone worker in `server/mqtt/mqtt_worker.py`.
2. Render uses `render.yaml`, boots Django from `server/dashboard/core/wsgi.py`, and starts `server/dashboard/core/mqtt_runtime.py` in-process.
3. Django publishes MQTT commands from `server/dashboard/core/apps/greenhouses/utils.py`.
4. MQTT feature flags live in `server/dashboard/core/settings.py`.
